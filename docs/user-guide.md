# Milo 知识库 Agent 用户指南

## 简介

Milo 是一个基于知识库的智能问答系统，可以帮助您快速检索和利用历史文档中的信息。

## 快速开始

### 1. 启动服务

```bash
# 后端
cd backend
pip install -e .
uvicorn app.main:app --reload

# 前端
cd frontend
npm install
npm run dev
```

访问 http://localhost:3000

### 2. 创建知识库

1. 点击左侧菜单"知识库"
2. 点击"创建知识库"按钮
3. 输入名称和描述
4. 点击"创建"

### 3. 上传文档

1. 进入知识库详情页
2. 点击"上传文档"
3. 选择 Markdown 或 PDF 文件
4. 点击"上传"
5. 点击"处理"按钮开始解析

### 4. 开始对话

1. 点击左侧菜单"对话"
2. 选择知识库（可选）
3. 输入问题
4. 获取基于知识库的回答

## 功能详解

### 知识库管理

- **创建**: 支持多个知识库，按主题分类
- **编辑**: 修改名称和描述
- **删除**: 删除知识库及其所有文档

### 文档管理

- **支持格式**: Markdown (.md), PDF (.pdf)
- **处理流程**: 解析 → 智能切片 → 向量化
- **状态说明**:
  - 待处理: 文档已上传，等待处理
  - 处理中: 正在解析和向量化
  - 已完成: 处理完成，可用于检索
  - 失败: 处理失败，可重试

### 智能对话

- **混合检索**: 向量相似度 + BM25 全文检索
- **Rerank 精排**: 提升检索结果相关性
- **引用溯源**: 显示答案来源，支持原文查看
- **流式响应**: 实时显示回答内容

### 对话历史

- 自动保存所有对话记录
- 支持搜索历史对话
- 可删除不需要的对话

## 最佳实践

### 文档准备

1. **结构清晰**: 使用 Markdown 标题组织内容
2. **内容完整**: 每个章节包含完整信息
3. **避免重复**: 相同内容只保留一份

### 提问技巧

1. **具体明确**: "如何处理合同签约超时？" 比 "签约问题" 更好
2. **包含关键词**: 使用专业术语
3. **分步提问**: 复杂问题拆分为多个小问题

### 知识库维护

1. **定期更新**: 及时上传新文档
2. **清理冗余**: 删除过时或重复的文档
3. **分类管理**: 按主题创建不同知识库

## 常见问题

### Q: 文档处理失败怎么办？

A: 检查文件格式是否正确，点击"重试"按钮重新处理。

### Q: 回答不准确怎么办？

A: 
1. 检查知识库中是否有相关文档
2. 尝试换一种问法
3. 确保文档内容准确完整

### Q: 如何提升检索效果？

A:
1. 上传高质量的文档
2. 使用清晰的标题结构
3. 避免文档内容重复
