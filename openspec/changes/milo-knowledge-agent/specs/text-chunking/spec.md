## 新增需求

### 需求：基于文档结构的智能切片

系统 SHALL 自动识别文档结构并进行智能切片，无需用户选择切片策略。

#### 场景：按标题分割（默认策略）

- **WHEN** 文档包含 Markdown 标题（## 或 #）
- **THEN** 系统按标题将文档分割为多个 section，每个 section 作为一个独立片段

#### 场景：超长 section 二次切分

- **WHEN** 单个 section 的 Token 数超过 max_chunk_size（默认 1200）
- **THEN** 系统在 section 内部按固定 Token 大小进行二次切分，保留 overlap（默认 100 token）

#### 场景：无标题文档回退

- **WHEN** 文档不包含任何标题结构
- **THEN** 系统回退到固定 Token 大小切片（chunk_size=1200, overlap=100）

#### 场景：处理短 section

- **WHEN** section 的 Token 数少于 min_chunk_size（默认 100）
- **THEN** 系统将该 section 与相邻 section 合并，避免产生过小的片段

#### 场景：保持上下文连续性

- **WHEN** section 被二次切分
- **THEN** 相邻片段共享约 100 个 Token 以保持上下文连续性

### 需求：保留片段来源信息

系统 SHALL 为每个片段维护来源信息。

#### 场景：追踪片段位置

- **WHEN** 文本被切分
- **THEN** 每个片段记录其在原始文档中的起始偏移量、结束偏移量和片段索引

#### 场景：追踪片段来源文档

- **WHEN** 文本被切分
- **THEN** 每个片段引用其来源文档 ID

### 需求：支持可配置的切片参数

系统 SHALL 允许配置切片行为，但用户无需选择切片策略。

#### 场景：自定义最大片段大小

- **WHEN** 用户指定自定义 max_chunk_size 参数
- **THEN** 系统使用指定的最大片段大小进行二次切分（默认 1200 token）

#### 场景：自定义重叠大小

- **WHEN** 用户指定自定义 overlap 参数
- **THEN** 系统使用指定的重叠大小进行切分（默认 100 token）

#### 场景：自定义最小片段大小

- **WHEN** 用户指定自定义 min_chunk_size 参数
- **THEN** 系统使用指定的最小片段大小进行 section 合并（默认 100 token）

### 需求：生成唯一片段标识符

系统 SHALL 为每个片段生成唯一标识符。

#### 场景：片段 ID 生成

- **WHEN** 片段被创建
- **THEN** 系统基于文档 ID 和片段索引生成确定性的片段 ID

#### 场景：片段 ID 唯一性

- **WHEN** 多个文档被处理
- **THEN** 所有片段 ID 在系统中保持唯一
